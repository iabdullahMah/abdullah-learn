How this site works.

abdullah-learn is Ag’s personal offensive-security notebook. Every page here was written to answer a specific question, usually after something surprising showed up in a pentest. No textbooks, no theory for theory’s sake — just the parts worth remembering.

What lives here

Short, opinionated explainers on web, API, auth, mobile, code review, enterprise architecture, tools, hardware, and DevSecOps. If a topic isn’t here yet, it hasn’t come up. When it does, Jarvis writes it up and it lands on the next push.

How a page gets here

  1. Question. You ask Jarvis to explain something.
  2. Page built. Jarvis writes an HTML file under /pages/ and updates manifest.json in the same commit.
  3. Push to GitHub.
  4. Penforge pulls. Webhook verifies signature, runs git pull --ff-only.
  5. Index updates. Reads manifest on load, no extra work.

How to read it

Nine categories, stable order:

  • web XSS, SQLi, SSRF, the OWASP workhorses.
  • api REST & GraphQL: BOLA, BFLA, mass assignment.
  • auth JWT, OAuth, SAML, session mechanics.
  • mobile iOS / Android runtime attacks, Frida, objection.
  • review White-box reading, language sinks.
  • enterprise WebSphere, DataPower, F5, WAFs, DMZ.
  • tools Burp, sqlmap, proxy chains, WAF bypass.
  • hardware UART, SPI, JTAG, BLE, firmware.
  • devsecops CI/CD, secrets, container escapes, IaC.

mobile tip: tap a chip to filter; tap it again to clear. pages marked “interactive” have clickable diagrams.

This is a notebook, not a reference.

Entries are opinionated, abbreviated, and reflect one person’s working memory. If something looks wrong or out of date, it probably is — and it stays wrong until Ag hits it again in the field.
